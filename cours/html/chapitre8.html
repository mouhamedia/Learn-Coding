Chapitre 8 : Les balises sémantiques 🏷️

Structure ton site avec des balises qui ont du sens !

🔹 Qu’est-ce que la sémantique ?

Les balises sémantiques décrivent le rôle du contenu (ex. : en-tête, navigation, article), ce qui améliore l’accessibilité et le référencement (SEO).

Analogie : Les balises sémantiques sont comme des étiquettes sur des dossiers : elles indiquent clairement ce qu’il y a dedans.

🔹 Balises sémantiques courantes

  • <header> : En-tête de la page ou d’une section.
  • <nav> : Menu de navigation.
  • <article> : Contenu indépendant (ex. : billet de blog).
  • <section> : Section thématique.
  • <footer> : Pied de page.

💻 Exemple pratique

Voici une page sémantique :


<!DOCTYPE html>
<html lang="fr">
<head>
  <meta charset="UTF-8">
  <title>Page sémantique</title>
</head>
<body>
  <header>
    <h1>Mon blog</h1>
  </header>
  <nav>
    <a href="#home">Accueil</a>
    <a href="#about">À propos</a>
  </nav>
  <article>
    <h2>Mon article</h2>
    <p>Ceci est un article intéressant.</p>
  </article>
  <footer>
    <p>© 2025 Mon blog</p>
  </footer>
</body>
</html>

Cet exemple utilise des balises sémantiques pour structurer une page.

📌 Points clés

  • Les balises sémantiques améliorent la lisibilité et le SEO.
  • Utilisez <header>, <nav>, etc., pour clarifier la structure.
  • Évitez les balises génériques comme <div> quand une balise sémantique convient.

🚀 Pour aller plus loin

Refactorise une page non sémantique en utilisant <section> et <aside>.
