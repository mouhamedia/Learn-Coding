Chapitre 4 : Encapsulation 🔒

Protège les données de tes objets pour un code plus sûr et clair !

🔹 Qu’est-ce que l’encapsulation ?

L’encapsulation consiste à regrouper les données (attributs) et les méthodes dans une classe, tout en contrôlant l’accès aux données. En Python, on utilise des attributs privés (avec __) pour les protéger.

🔹 Pourquoi utiliser l’encapsulation ?

  • Protéger les données contre des modifications non désirées.
  • Rendre le code plus clair en définissant des interfaces claires (getters/setters).

💻 Exemple en Python

Voici une classe CompteBancaire avec un solde privé :


class CompteBancaire:
    def __init__(self, titulaire, solde_initial):
        self.titulaire = titulaire
        self.__solde = solde_initial  # Attribut privé
    
    def deposer(self, montant):
        if montant > 0:
            self.__solde += montant
            print(f"Nouveau solde : {self.__solde}")
    
    def retirer(self, montant):
        if montant <= self.__solde:
            self.__solde -= montant
            print(f"Nouveau solde : {self.__solde}")
        else:
            print("Solde insuffisant.")
    
    def get_solde(self):
        return self.__solde

compte = CompteBancaire("Alice", 1000)
compte.deposer(500)      # Affiche : Nouveau solde : 1500
compte.retirer(200)      # Affiche : Nouveau solde : 1300
print(compte.get_solde())  # Affiche : 1300
# print(compte.__solde)   # Erreur : attribut privé

📌 Points clés

  • Utilise __ pour rendre un attribut privé.
  • Les getters (ex. : get_solde) permettent d’accéder aux données.
  • Les setters contrôlent les modifications.
