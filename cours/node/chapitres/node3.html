Chapitre 3 : Modules natifs

Explore les modules natifs de Node.js pour étendre les fonctionnalités de tes scripts.

Qu’est-ce qu’un module natif ?

Node.js fournit des modules natifs (comme fs, path, os) qui offrent des fonctionnalités prêtes à l’emploi sans installation supplémentaire.

Module path

Le module path aide à manipuler les chemins de fichiers de manière portable.

const path = require('path');

const filePath = path.join('folder', 'subfolder', 'file.txt');
console.log('Chemin :', filePath);

const fileName = path.basename(filePath);
console.log('Nom du fichier :', fileName);

Sortie (sur Windows, par exemple) :

Chemin : folder\subfolder\file.txt
Nom du fichier : file.txt

Module os

Le module os fournit des informations sur le système d’exploitation.

const os = require('os');

console.log('Plateforme :', os.platform());
console.log('Mémoire libre :', os.freemem() / 1024 / 1024, 'MB');
console.log('Utilisateur :', os.userInfo().username);

Sortie (exemple) :

Plateforme : win32
Mémoire libre : 4096 MB
Utilisateur : jean

Exemple pratique : Informations système

Voici un script qui utilise path et os pour afficher des informations système :

const path = require('path');
const os = require('os');

const homeDir = os.homedir();
const configPath = path.join(homeDir, 'config', 'settings.json');

console.log('Répertoire personnel :', homeDir);
console.log('Chemin du fichier de configuration :', configPath);
console.log('Système :', os.type());
console.log('Mémoire totale :', os.totalmem() / 1024 / 1024, 'MB');

Sortie (exemple) :

Répertoire personnel : /home/jean
Chemin du fichier de configuration : /home/jean/config/settings.json
Système : Linux
Mémoire totale : 8192 MB

Bonnes pratiques :

  • Utilise path.join pour créer des chemins compatibles avec tous les systèmes.
  • Explore la documentation officielle de Node.js pour découvrir d’autres modules natifs.
  • Évite de réinventer des fonctionnalités déjà disponibles dans les modules natifs.
