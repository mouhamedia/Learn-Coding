Chapitre 3 : JSX et rendu 📄

Maîtrise JSX, la syntaxe qui combine HTML et JavaScript, pour créer des interfaces utilisateur dynamiques.

🔹 Qu'est-ce que JSX ?

JSX (JavaScript XML) est une extension de syntaxe qui permet d’écrire du code semblable à HTML dans JavaScript. Il est utilisé par React pour décrire l’interface utilisateur de manière déclarative, rendant le code plus lisible et maintenable.

Caractéristiques principales :

  • Syntaxe HTML-like: Écriture intuitive, comme <div>Bonjour</div>.
  • Expressions JavaScript: Intègre des variables ou calculs avec {}.
  • Virtual DOM: JSX est converti en appels JavaScript pour mettre à jour l’interface efficacement.

🔹 Pourquoi utiliser JSX ?

JSX simplifie la création d’interfaces en combinant la logique et la présentation. Il est compilé par des outils comme Babel en code JavaScript pur, compatible avec le Virtual DOM de React.

Règles de JSX :

  • Chaque élément doit avoir un parent unique.
  • Utilise className au lieu de class et htmlFor au lieu de for.
  • Les balises doivent être fermées (ex. <img />).

🔹 Exemple 1 : JSX avec expressions

Affiche des données dynamiques avec JSX :


// src/App.js
import React from 'react';

function App() {
  const user = { name: 'Alice', age: 25 };
  return (
    

      
Salut, {user.name} !

      
Âge : {user.age}

    

  );
}

export default App;

Explication:

  • {user.name}: Insère la valeur de la variable dans le JSX.
  • <div>: Parent unique requis pour le rendu.

Résultat: Affiche “Salut, Alice !” et “Âge : 25” à http://localhost:3000.

🔹 Exemple 2 : Rendu conditionnel

Affiche un message selon une condition :


// src/App.js
import React from 'react';

function App() {
  const isLoggedIn = true;
  return (
    

      
Statut

      {isLoggedIn ? 
Connecté
 : 
Déconnecté
}
    

  );
}

export default App;

Résultat: Affiche “Connecté”.

🔹 Exemple 4 : Styles inline

Applique des styles avec JSX :


// src/App.js
import React from 'react';

function App() {
  const style = { color: 'blue', fontSize: '20px' };
  return (
    

      
Titre

      
Texte stylé

    

  );
}

export default App;

Résultat: Affiche un titre rouge et un texte bleu.

🔹 Exemple 5 : Fragments

Utilise des fragments pour éviter des divs inutiles :


// src/App.js
import React from 'react';

function App() {
  return (
    <>
      
Titre

      
Paragraphe

    
  );
}

export default App;

Résultat: Affiche “Titre” et “Paragraphe” sans div englobant.

🔹 Erreurs courantes à éviter

  • Utiliser class au lieu de className: JSX utilise les conventions JavaScript.
  • Oublier la clé dans les listes: Ajoute key dans map() pour les performances.
  • Balises non fermées: Assure-toi que chaque balise est fermée (ex. <img />).
