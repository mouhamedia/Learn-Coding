Chapitre 3 : Création d’un projet Django 🏗️

Apprends à créer et configurer un projet Django pour démarrer ton développement.

🔹 Qu’est-ce qu’un projet Django ?

Un projet Django est un ensemble de configurations et d’applications qui forment une application web. Il inclut les fichiers de configuration globale (comme la base de données et les URLs) et peut contenir plusieurs applications pour des fonctionnalités spécifiques.

Différence projet vs application :

  • Projet : Conteneur global (ex. un site web complet).
  • Application : Module spécifique (ex. blog, boutique) réutilisable dans plusieurs projets.

🔹 Exemple 1 : Créer un projet

Dans ton environnement virtuel activé, crée un projet :


django-admin startproject monprojet

Résultat : Crée un dossier monprojet avec :


monprojet/
├── manage.py
├── monprojet/
│   ├── __init__.py
│   ├── settings.py
│   ├── urls.py
│   ├── wsgi.py

Explication :

  • manage.py : Script pour gérer le projet (serveur, migrations, etc.).
  • settings.py : Configure la base de données, les applications installées, et les paramètres.
  • urls.py : Définit les routes du projet.

🔹 Exemple 2 : Lancer le serveur

Accède au dossier du projet et lance le serveur de développement :


cd monprojet
python manage.py runserver

Résultat : Ouvre http://localhost:8000 pour voir la page par défaut de Django.

Exemple alternatif : Si le port 8000 est occupé :


python manage.py runserver 8001

Résultat : Serveur accessible à http://localhost:8001.

🔹 Exemple 3 : Configurer la base de données

Par défaut, Django utilise SQLite. Vérifie la configuration dans monprojet/settings.py :


DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.sqlite3',
        'NAME': BASE_DIR / 'db.sqlite3',
    }
}

Explication : SQLite est une base légère stockée dans db.sqlite3. Pour une base MySQL ou PostgreSQL, modifie ainsi (exemple avec MySQL) :


DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.mysql',
        'NAME': 'mabase',
        'USER': 'root',
        'PASSWORD': '',
        'HOST': 'localhost',
        'PORT': '3306',
    }
}

Installe le pilote MySQL :


pip install mysqlclient

Résultat : Le projet est prêt à utiliser MySQL après configuration.

🔹 Exemple 4 : Appliquer les migrations initiales

Django inclut des migrations pour les fonctionnalités de base (ex. utilisateurs). Applique-les :


python manage.py migrate

Résultat : Crée la base db.sqlite3 et les tables nécessaires (ex. auth_user pour les utilisateurs).

🔹 Erreurs courantes à éviter

  • Commande exécutée hors du dossier : Assure-toi d’être dans le dossier contenant manage.py.
  • Base de données mal configurée : Vérifie settings.py et installe les pilotes nécessaires.
  • Permissions manquantes : Accorde les droits d’écriture au dossier du projet si SQLite échoue.

📝 Exercice

4. Applique les migrations initiales avec python manage.py migrate.
5. Crée un fichier texte décrivant la structure du dossier blog.
