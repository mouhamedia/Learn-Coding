Chapitre 1 : Introduction à Vue.js

Découvre les bases de Vue.js, un framework JavaScript moderne pour créer des interfaces utilisateur.

Qu’est-ce que Vue.js ?

Vue.js est un framework JavaScript progressif pour construire des interfaces utilisateur. Il est conçu pour être simple à adopter, flexible, et performant, tout en offrant une approche réactive pour créer des applications web dynamiques.

Pourquoi utiliser Vue.js ?

Vue.js est apprécié pour :

  • Sa simplicité : Facile à apprendre pour les débutants.
  • Sa réactivité : Les changements dans les données se reflètent automatiquement dans l’interface.
  • Sa modularité : Peut être utilisé pour des projets simples ou complexes.
  • Sa communauté : Une vaste communauté et des outils comme Vue Router et Vuex.

Installer Vue.js

Pour commencer avec Vue 3, tu peux utiliser un CDN pour des projets simples ou Vite pour des projets modernes.

Avec un CDN :

<script src="https://unpkg.com/vue@3/dist/vue.global.js"></script>

Avec Vite, exécute :

npm create vite@latest mon-projet -- --template vue
cd mon-projet
npm install
npm run dev

Premier exemple : Hello Vue

Voici un exemple simple d’une application Vue.js utilisant le CDN :

<!DOCTYPE html>
<html>
<head>
  <title>Hello Vue</title>
  <script src="https://unpkg.com/vue@3/dist/vue.global.js"></script>
</head>
<body>
  <div id="app">
    <p>{{ message }}</p>
  </div>
  <script>
    Vue.createApp({
      data() {
        return {
          message: 'Hello Vue.js !'
        };
      }
    }).mount('#app');
  </script>
</body>
</html>

Cet exemple affiche un message réactif dans le DOM en utilisant la méthode createApp de Vue 3.

Structure d’un composant Vue

Un composant Vue combine HTML, JavaScript, et CSS. Voici un exemple de composant dans un fichier .vue :

<template>
  <div>
    <h1>{{ greeting }}</h1>
  </div>
</template>
<script>
export default {
  data() {
    return {
      greeting: 'Bienvenue dans Vue.js !'
    };
  }
};
</script>
<style scoped>
h1 {
  color: #42b883;
}
</style>

Ce composant utilise la syntaxe des Single-File Components (SFC), avec un template, un script, et du CSS scopé.

Bonnes pratiques :

  • Utilise Vite ou un autre bundler pour les projets réels, car les CDN sont mieux adaptés aux prototypes.
  • Commence avec l’Options API pour sa simplicité avant d’explorer la Composition API.
  • Assure-toi que ton environnement Node.js est à jour pour éviter les problèmes de compatibilité.
