Chapitre 3 : Structure d’un composant

Comprendre les bases d’un composant Vue.js et son organisation.

Qu’est-ce qu’un composant Vue.js ?

Un composant Vue.js est une unité réutilisable qui encapsule une partie de l’interface utilisateur (UI) et sa logique. Les composants sont au cœur de Vue.js, permettant de construire des applications modulaires et maintenables. Chaque composant peut inclure du HTML, du CSS et du JavaScript, organisés de manière cohérente.

Les trois parties principales d’un composant

Dans Vue 3, un composant est généralement défini dans un fichier avec l’extension .vue (Single-File Component, SFC). Un SFC est divisé en trois sections :

  • Template : Contient le HTML et les directives Vue pour décrire l’interface.
  • Script : Contient la logique JavaScript (Options API ou Composition API).
  • Style : Contient le CSS pour styliser le composant.

Exemple d’un composant simple

Voici un exemple de composant Vue.js nommé HelloWorld.vue :

<template>
  <div>
    <h1>{{ message }}</h1>
    <button @click="updateMessage">Changer le message</button>
  </div>
</template>

<script>
export default {
  name: 'HelloWorld',
  data() {
    return {
      message: 'Bienvenue dans Vue.js !'
    };
  },
  methods: {
    updateMessage() {
      this.message = 'Vous avez cliqué sur le bouton !';
    }
  }
};
</script>

<style scoped>
h1 {
  color: #42b883;
}
button {
  padding: 0.5rem 1rem;
  background-color: #2a9a78;
  color: white;
  border: none;
  border-radius: 4px;
  cursor: pointer;
}
button:hover {
  background-color: #1b684f;
}
</style>

Explication des sections

Template : La section <template> définit la structure HTML. Ici, on utilise une interpolation {{ message }} pour afficher une donnée dynamique et une directive @click pour lier un événement à une méthode.

Script : La section <script> utilise l’Options API. On définit :

  • name : Le nom du composant (optionnel, mais utile pour le débogage).
  • data : Une fonction qui retourne les données réactives du composant.
  • methods : Les fonctions qui gèrent les interactions, comme updateMessage.

Style : La section <style> avec l’attribut scoped garantit que le CSS s’applique uniquement à ce composant, évitant les conflits avec d’autres parties de l’application.

Astuce : L’attribut scoped dans la balise <style> est une fonctionnalité puissante de Vue.js. Il génère des sélecteurs uniques pour éviter que le CSS d’un composant n’affecte d’autres composants.

Exemple pratique : Compteur simple

Voici un exemple de composant qui implémente un compteur :

<template>
  <div>
    <p>Compteur : {{ count }}</p>
    <button @click="increment">Incrémenter</button>
    <button @click="decrement">Décrémenter</button>
  </div>
</template>

<script>
export default {
  name: 'Counter',
  data() {
    return {
      count: 0
    };
  },
  methods: {
    increment() {
      this.count++;
    },
    decrement() {
      this.count--;
    }
  }
};
</script>

<style scoped>
p {
  font-size: 1.2rem;
  color: #333;
}
button {
  margin: 0.5rem;
  padding: 0.5rem 1rem;
  background-color: #42b883;
  color: white;
  border: none;
  border-radius: 4px;
}
button:hover {
  background-color: #2a9a78;
}
</style>

Dans cet exemple, le composant Counter utilise une donnée réactive count et deux méthodes pour incrémenter ou décrémenter cette valeur. Le template affiche la valeur actuelle et lie les boutons aux méthodes via la directive @click.

Bonnes pratiques :

  • Nommez vos composants de manière claire et descriptive (ex. : Counter, UserCard).
  • Utilisez toujours scoped pour le CSS sauf si vous avez besoin de styles globaux.
  • Structurez votre code de manière lisible en séparant bien les sections template, script, et style.
